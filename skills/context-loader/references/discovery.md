# 上下文发现（Context Discovery）

## 发现流程

Agent 启动时，按以下顺序发现上下文：

### 1. 读取 active.md

```
读取: 04-Working/active.md
目的: 了解当前活跃项目、任务、犹豫中的决策
输出: 当前关注点列表
```

### 2. 读取各层索引

```
读取: 03-Episodic/index.md    → 情景列表（标题+关键词+教训）
读取: 01-Procedural/index.md  → 方法列表（方法名+适用场景+有效性）
读取: 02-Semantic/Areas/      → 知识领域索引（如有）
```

### 3. 从用户任务提取关键词

```
输入: 用户的当前任务/问题
处理: 提取关键词（项目名、技术栈、人名、概念）
匹配: 在各层索引中查找匹配的条目
```

### 4. 匹配规则

| 匹配类型 | 说明 | 示例 |
|---------|------|------|
| 关键词匹配 | 标题/标签中包含关键词 | "ASP" → 找到 ASP 相关情景 |
| 场景匹配 | 用户说"做个类似 X 的" → 查找涉及 X 的事件 | "再做个类似需求确认的" → 找需求确认相关情景 |
| 方法匹配 | 用户问"用什么方法好" → 查找相关方法 | "怎么提炼知识" → 找分级沉淀规则 |

### 匹配输出

```
检索到相关历史：

**相关情景：**
- [[情景名]] — 关键词摘要；核心教训
- [[情景名]] — ...

**相关方法：**
- [[方法名]] — 适用场景；有效性状态
- [[方法名]] — ...

需要我加载哪些全文？
```
